# Пайплайн изображений

## Что делает скрипт

Скрипт `scripts/optimize-images.js`:

- берет исходники из `assets/images/source`
- конвертирует каждый файл в `webp`
- создает несколько размеров в отдельных папках
- может работать в режиме слежения за папкой (`watch`)

## Требования

- Установленный `ImageMagick` (команда `magick`).
- В вашем окружении сейчас доступен `magick` из ServBay, поэтому скрипт работает без дополнительных npm-пакетов.

## Папки результата

Результаты складываются в `assets/images/webp`:

- `original-webp` - оригинал в webp без ресайза
- `hero-800x480`
- `about-800x520`
- `card-600x400`
- `thumb-320x220`
- `og-1200x630`

Структура подпапок внутри `source` сохраняется.

## Какой размер куда вставлять

- `hero` (главные баннеры): `800x480`
  Папка: `assets/images/webp/hero-800x480`
- `about` (блок "О нас"): `800x520`
  Папка: `assets/images/webp/about-800x520`
- `card` (карточки/внутренние блоки): `600x400`
  Папка: `assets/images/webp/card-600x400`
- `thumb` (миниатюры/превью): `320x220`
  Папка: `assets/images/webp/thumb-320x220`
- `og` (превью для соцсетей): `1200x630`
  Папка: `assets/images/webp/og-1200x630`

## Рекомендации по исходникам

- Загружайте исходник минимум в 1.5-2 раза больше целевого размера.
- Практический минимум:
  - для `hero` и `about`: от `1600px` по ширине
  - для `og`: от `2400x1260`

## Команды

Установить зависимости:

```bash
npm install
```

Разовая оптимизация:

```bash
npm run images:optimize
```

Слежение за папкой (автоконвертация при добавлении/обновлении файлов):

```bash
npm run images:watch
```

Принудительная регенерация всех вариантов:

```bash
npm run images:force
```

## Поддерживаемые форматы входа

`.jpg`, `.jpeg`, `.png`, `.webp`, `.avif`, `.tif`, `.tiff`
