# Руководство по исправлению HTTP → HTTPS

## Проблема
При сканировании сайта обнаружены URL, использующие небезопасный протокол HTTP вместо HTTPS.

## Что было сделано

### 1. Добавлен редирект в .htaccess
В файл `.htaccess` добавлено правило принудительного редиректа с HTTP на HTTPS:

```apache
# Принудительный редирект с HTTP на HTTPS
<IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteCond %{HTTPS} off
    RewriteRule ^(.*)$ https://%{HTTP_HOST}/$1 [R=301,L]
</IfModule>
```

Это правило:
- Проверяет, если соединение не защищено (HTTPS off)
- Перенаправляет все запросы на HTTPS-версию с кодом 301 (постоянный редирект)
- Сохраняет имя хоста и путь к странице

### 2. Проверка внутренних ссылок
Все внутренние ссылки в файлах проекта используют:
- Относительные пути (например, `/services.html`, `/about.html`)
- Абсолютные HTTPS URL (например, `https://zamok-i.ru/`)

**HTTP-ссылок в коде не обнаружено.**

### 3. Sitemap.xml
Файл `sitemap.xml` уже содержит HTTPS-версии всех URL.

## Что нужно сделать на сервере

### Шаг 1: Загрузить обновленный .htaccess
```bash
# Скопируйте файл .htaccess на сервер
scp .htaccess user@server:/path/to/site/
```

Или через FTP/SFTP клиент.

### Шаг 2: Проверить работу редиректа
После загрузки проверьте:

1. **Проверьте редирект с HTTP на HTTPS:**
   ```bash
   curl -I http://zamok-i.ru
   ```
   Ожидаемый ответ:
   ```
   HTTP/1.1 301 Moved Permanently
   Location: https://zamok-i.ru/
   ```

2. **Проверьте отдельную страницу:**
   ```bash
   curl -I http://zamok-i.ru/service/zamena/zamena-zamka-dveri.html
   ```
   Должен быть редирект на HTTPS-версию.

### Шаг 3: Проверить mod_rewrite
Убедитесь, что на сервере включен модуль `mod_rewrite` для Apache:

```bash
# Проверка включенного модуля (если есть SSH доступ)
apache2ctl -M | grep rewrite
```

Если модуль не включен, включите его или обратитесь к хостинг-провайдеру.

### Шаг 4: Обновить внешние ссылки
Если на сайте есть ссылки на внешние ресурсы, использующие HTTP, замените их на HTTPS:

- Изображения
- Скрипты
- Стили
- iframe

### Шаг 5: Проверить Google Search Console
После внедрения:
1. Отправьте обновленный `sitemap.xml` в Google Search Console
2. Используйте инструмент проверки URL для проверки редиректов
3. Отслеживайте отчет "Покрытие" для мониторинга индексации HTTPS-страниц

## Дополнительные проверки

### Проверка смешанного контента
Откройте консоль разработчика в браузере (F12) и проверьте, нет ли предупреждений о смешанном контенте (mixed content).

### Проверка всех страниц
Используйте инструменты для сканирования:
- **Screaming Frog SEO Spider**
- **Sitebulb**
- **Google Chrome Lighthouse**

Убедитесь, что все страницы возвращают HTTPS.

## Примечания

- HSTS заголовок уже настроен в `.htaccess` (1 год)
- Все canonical URL используют HTTPS
- Все Open Graph и Twitter Card метатеги используют HTTPS
- Schema.org разметка использует HTTPS

## Контакты для вопросов
Если возникнут проблемы при внедрении, обратитесь к хостинг-провайдеру или проверьте документацию Apache по mod_rewrite.
