# Исправление HTTP → HTTPS: Краткая инструкция

## ✅ Что уже сделано

1. **Добавлен редирект в .htaccess**
   - Все HTTP-запросы будут перенаправляться на HTTPS (301 редирект)
   - HSTS заголовок уже настроен

2. **Проверены все файлы проекта**
   - Все внутренние ссылки используют HTTPS или относительные пути
   - Sitemap.xml содержит HTTPS-версии
   - Нет HTTP-ссылок в коде

## 🚀 Что нужно сделать

### 1. Загрузить .htaccess на сервер
```bash
# Через rsync (если используете)
rsync -avz .htaccess user@server:/path/to/site/

# Или через scp
scp .htaccess user@server:/path/to/site/
```

### 2. Проверить редирект
```bash
# Проверка главной страницы
curl -I http://zamok-i.ru

# Ожидаемый ответ:
# HTTP/1.1 301 Moved Permanently
# Location: https://zamok-i.ru/

# Проверка страницы услуги
curl -I http://zamok-i.ru/service/zamena/zamena-zamka-dveri.html
```

### 3. Убедиться, что mod_rewrite включен
На большинстве хостингов Apache mod_rewrite включен по умолчанию.
Если нет — обратитесь к хостинг-провайдеру.

### 4. Проверить в Google Search Console
- Отправьте sitemap.xml повторно
- Проверьте отчет "Покрытие"
- Убедитесь, что индексируются HTTPS-версии

## 📋 Проверка локально

Перед загрузкой проверьте проект на HTTP-ссылки:

```bash
npm run check:http
```

Если всё хорошо, увидите:
```
✅ HTTP-ссылок не найдено. Все ссылки используют HTTPS или относительные пути.
```

## ❓ Вопросы

Если что-то не работает:
1. Проверьте логи Apache/Nginx
2. Убедитесь, что SSL-сертификат установлен и действителен
3. Проверьте, что сайт доступен по HTTPS напрямую
